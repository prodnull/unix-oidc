Step-up authentication for Linux SSH and sudo with OIDC

🔐 DPoP Token Binding 📱 Device Flow MFA 🦀 Memory-Safe Rust 📊 Structured Audit Logs
1
OIDC Login Flow
Standard OpenID Connect authorization code flow for SSH authentication.
  • User authenticates with their existing identity provider
  • Username maps to Unix account via SSSD/LDAP
  • Token issued with DPoP cryptographic binding
2
Sudo Step-Up Authentication
OAuth 2.0 Device Authorization Grant (RFC 8628) for privileged command approval.
  • PAM module intercepts sudo commands requiring step-up
  • Device code displayed in terminal for mobile approval
  • User approves on phone/browser, command executes
3
Policy Configuration
YAML-based policy configuration for fine-grained access control.
  • Host classification: standard, elevated, critical
  • Command classes for grace period grouping
  • Per-command overrides for sensitive operations
4
Audit Events
Structured JSON audit logging for security monitoring and compliance.
  • All auth events logged with session correlation IDs
  • SIEM-ready format (Splunk, ELK, etc.)
  • Grace periods, denials, and timeouts tracked
