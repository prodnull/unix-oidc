DPoP-Bound OIDC Authentication

Tokens that are useless if stolen

We implemented RFC 9449 (DPoP) for cryptographic token binding. Each token is mathematically bound to your device's private key.

Why This Matters

  • Token intercepted in transit? Useless.
  • Token extracted from memory? Useless.
  • Token found in logs? Useless.
  • Each proof expires in 60 seconds
  • Each proof is bound to one specific server

Get Started

Download Latest Release

# 1. Download (tarball + signature + certificate)
export V=$(curl -s https://api.github.com/repos/prodnull/unix-oidc/releases/latest | grep tag_name | cut -d'"' -f4)
export P=linux-x86_64
curl -LO https://github.com/prodnull/unix-oidc/releases/download/$V/unix-oidc-$V-$P.tar.gz
curl -LO https://github.com/prodnull/unix-oidc/releases/download/$V/unix-oidc-$V-$P.tar.gz.sig
curl -LO https://github.com/prodnull/unix-oidc/releases/download/$V/unix-oidc-$V-$P.tar.gz.pem
# 2. Verify signature (optional but recommended)
cosign verify-blob \
  --certificate unix-oidc-$V-$P.tar.gz.pem \
  --signature unix-oidc-$V-$P.tar.gz.sig \
  --certificate-identity-regexp 'https://github.com/prodnull/unix-oidc' \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  unix-oidc-$V-$P.tar.gz
# 3. Install
tar -xzf unix-oidc-$V-$P.tar.gz
sudo cp libpam_unix_oidc.so /usr/lib/security/
