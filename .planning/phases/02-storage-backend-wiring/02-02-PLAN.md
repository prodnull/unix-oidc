  <name>Task 1: Implement maybe_migrate() with atomic rollback and unit tests</name>
  <files>unix-oidc-agent/src/storage/router.rs</files>
  <behavior>
    - Test: migration moves all 4 keys from file to keyring when all exist (mock backend)
    - Test: migration returns Migrated(4) and logs INFO message
    - Test: migration verifies read-back from keyring matches original value for each key
    - Test: migration rollback when key 3 write fails: keys 1+2 deleted from keyring, returns Err
    - Test: migration rollback when key 2 read-back verification fails: key 1 deleted from keyring
    - Test: migration returns NotApplicable (0 keys) when no file credentials exist
    - Test: migration calls file.delete() for each migrated key (secure delete path)
    - Test: migration only runs when router.kind is NOT BackendKind::File (no file-to-file migration)
  </behavior>
  <action>
    Add `maybe_migrate()` as a method on `StorageRouter` (or a free function taking `&StorageRouter` and `&FileStorage`). Implementation per CONTEXT.md decisions:

    1. If `self.kind` is `BackendKind::File`, set `migration_status = NotApplicable` and return Ok(0).
    2. Create a `FileStorage::new()` to check for existing file credentials.
    3. Collect all 4 keys (`KEY_DPOP_PRIVATE`, `KEY_ACCESS_TOKEN`, `KEY_REFRESH_TOKEN`, `KEY_TOKEN_METADATA`) that exist in file storage.
    4. If none exist, set `migration_status = NotApplicable`, return Ok(0).
    5. For each key: write to keyring via `self.backend.store()`, then read back via `self.backend.retrieve()` and compare. If mismatch or error, call `rollback_migration()` to delete all previously-migrated keys from keyring, return Err.
    6. After all keys migrated and verified: call `file.delete(key)` for each (uses secure_remove from Phase 1). Log best-effort failures at WARN per key.
    7. Set `migration_status = Migrated(count)`, log `info!("migrated {} credentials from file storage to keyring", count)`.

    `rollback_migration()`: iterate previously-migrated keys, call `self.backend.delete()` for each. Log WARN on individual rollback failures but continue rolling back remaining keys. Per Pitfall 6 in research: treat rollback failure as best-effort.

    Unit tests should use `keyring::mock` backend (via `set_default_credential_builder`) and `FileStorage::with_base_dir(tempdir)` for the file source.

    **Step 2 — Replace 7 FileStorage::new() call sites:**

    At each site, replace `FileStorage::new()` with `StorageRouter::detect()`. The `StorageRouter` implements `SecureStorage` so it is a drop-in replacement. Specific sites:

    1. `main.rs:242` (run_status non-daemon): `if let Ok(storage) = StorageRouter::detect() {` — this path checks local files when daemon isn't running. StorageRouter handles this correctly.

    3. `main.rs:560` (run_refresh): `let storage = StorageRouter::detect()?;`
    4. `main.rs:595` (run_logout): `let storage = StorageRouter::detect()?;`
    5. `main.rs:730` (run_serve pre-daemon): `let storage = StorageRouter::detect()?;` — call `storage.maybe_migrate()` here since daemon startup is the primary migration trigger per CONTEXT.md.
    6. `main.rs:762` (run_serve daemon init): `let storage = match StorageRouter::detect() {` — same pattern as current FileStorage match.
    7. `socket.rs:404`: `let storage = StorageRouter::detect().map_err(|e| format!("Storage error: {}", e))?;`
