# Plan 16-03: Break-Glass Fallback + OIDC Failure Graceful Degradation

**Phase:** 16 — Rigorous Integration Testing (Gap Closure)
**Requirements:** INT-03
**Depends on:** Phase 8 (break-glass implementation)

## Goal

Verify that when the OIDC IdP is unavailable: (1) break-glass accounts can still authenticate, (2) normal OIDC auth fails gracefully without hanging or crashing, and (3) OIDC auth recovers when the IdP comes back online.

## Tasks

### Task 1: Break-Glass Fallback Integration Test

Create `test/tests/test_break_glass_fallback.sh`:

1. **Phase A — Baseline**: With Keycloak running:
   - Verify OIDC token acquisition works (direct grant to /token endpoint)
   - Confirm the break-glass account is configured in test policy.yaml

2. **Phase B — IdP Down**: Stop Keycloak container:
   - Attempt OIDC token acquisition → expect failure (connection refused / timeout)
   - Verify the PAM module's break-glass path works:
     * Call the PAM module's authenticate with break-glass user via test harness
     * OR: test the `is_break_glass_user()` logic via a Rust integration test that sets up policy config and verifies `PamError::IGNORE` return
   - Verify audit event is emitted for break-glass auth

3. **Phase C — IdP Recovery**: Restart Keycloak container:
   - Wait for health check
   - Verify OIDC token acquisition works again
   - Verify normal user authentication succeeds

### Task 2: PAM-Level Break-Glass Integration Test (Rust)

Create `pam-unix-oidc/tests/break_glass_integration.rs`:

1. Test with `UNIX_OIDC_TEST_MODE=true` and policy config:
   - Break-glass user configured
   - OIDC issuer set to unreachable URL (e.g., `http://127.0.0.1:1`)

2. Scenarios:
   - Break-glass user → returns IGNORE (passes to next PAM module)
   - Non-break-glass user with unreachable IdP → returns AUTH_ERR (not hang/crash)
   - Break-glass user with working IdP → still returns IGNORE (break-glass is unconditional)

3. Verify: no panic, no hang (timeout guard), correct error codes

### Task 3: CI Wiring

Add break-glass test to existing `integration` CI job:
- The shell test runs as part of docker-compose.test.yaml environment
- Add docker stop/start commands for Keycloak within the test

## Success Criteria

- [ ] Break-glass account authenticates when Keycloak is stopped
- [ ] Normal OIDC auth fails gracefully (no hang, no crash) when IdP is down
- [ ] OIDC auth recovers after Keycloak restart
- [ ] Rust integration test validates PAM error codes
- [ ] Break-glass audit event is emitted

## Security Considerations

- Break-glass bypass must ONLY apply to configured accounts — test verifies non-configured users are denied
- Break-glass path returns IGNORE (not SUCCESS) — next PAM module must still verify credentials
- Audit trail is mandatory for break-glass usage — test verifies event emission
