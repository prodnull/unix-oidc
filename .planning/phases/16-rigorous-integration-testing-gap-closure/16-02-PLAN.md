# Plan 16-02: Step-Up IPC Full-Flow Integration Test

**Phase:** 16 — Rigorous Integration Testing (Gap Closure)
**Requirements:** INT-02
**Depends on:** Plan 16-01 (CIBA realm), Phases 10, 13 (step-up + IPC)

## Goal

Build an integration test that exercises the complete step-up IPC round-trip: PAM sends StepUp → agent spawns CIBA poll → agent returns StepUpPending → PAM polls StepUpResult → StepUpComplete. Uses wiremock-rs to mock the CIBA IdP endpoints so the test runs without external dependencies.

## Tasks

### Task 1: Add wiremock-rs Dev Dependency

Add `wiremock = "0.6"` to `unix-oidc-agent/Cargo.toml` `[dev-dependencies]`.

### Task 2: Step-Up IPC Integration Test

Create `unix-oidc-agent/tests/step_up_ipc.rs`:

1. **Setup**: Start a wiremock MockServer that serves:
   - `GET /.well-known/openid-configuration` → OIDC discovery JSON with `backchannel_authentication_endpoint` and `token_endpoint` pointing to the mock
   - `POST /protocol/openid-connect/ciba-auth` → `{"auth_req_id":"test-123","expires_in":120,"interval":2}`
   - `POST /protocol/openid-connect/token` → first call returns `{"error":"authorization_pending"}`, second call returns `{"access_token":"tok","id_token":"...","token_type":"Bearer","expires_in":300}` with ACR claim

2. **Agent setup**: Spawn the `unix-oidc-agent` binary with env vars:
   - `UNIX_OIDC_OIDC_ISSUER` = wiremock server URL
   - `UNIX_OIDC_OIDC_CLIENT_ID` = `test-ciba`
   - `UNIX_OIDC_OIDC_CLIENT_SECRET` = `test-secret`
   - `UNIX_OIDC_STORAGE_BACKEND` = `file`
   - Custom socket path in tempdir

3. **IPC flow**:
   a. Send `StepUp { username: "testuser", command: "sudo systemctl restart", hostname: "test-host", method: "push", timeout_secs: 30 }`
   b. Expect `StepUpPending { correlation_id, expires_in, poll_interval_secs }`
   c. Wait `poll_interval_secs` seconds
   d. Send `StepUpResult { correlation_id }`
   e. If still pending, wait and retry
   f. Expect `StepUpComplete { acr, session_id }`

4. **Negative test — timeout**: Configure wiremock to always return `authorization_pending`; send StepUp with `timeout_secs: 5`; expect `StepUpTimedOut`

5. **Negative test — access denied**: Configure wiremock to return `{"error":"access_denied"}`; expect `StepUpTimedOut { reason: "access_denied" }`

6. **Cleanup**: Send Shutdown, wait for process exit

### Task 3: Verify Test Compiles and Passes

Run `cargo test -p unix-oidc-agent --test step_up_ipc` locally.

## Success Criteria

- [ ] wiremock-rs added to dev-dependencies
- [ ] Happy-path test: StepUp → StepUpPending → poll → StepUpComplete
- [ ] Timeout test: StepUp → StepUpPending → ... → StepUpTimedOut
- [ ] Access denied test: StepUp → StepUpTimedOut(access_denied)
- [ ] All tests pass with `cargo test`

## Security Considerations

- Mock server uses localhost-only binding (no external exposure)
- Test validates ACR claim presence in StepUpComplete response
- Timeout test confirms the agent doesn't hang indefinitely
