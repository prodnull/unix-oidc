<objective>
Fix the blocking defect where token refresh drops the `signer_type` field from stored metadata, causing hardware signer users to silently fall back to software signing after refresh + daemon restart.

Purpose: Without this fix, YubiKey and TPM users lose their DPoP hardware binding after any token refresh, silently degrading to software keys — breaking the core security guarantee of non-exportable hardware-bound DPoP.

Output: Two-line code fix in both refresh paths + regression test proving signer_type persistence across refresh.
</objective>
