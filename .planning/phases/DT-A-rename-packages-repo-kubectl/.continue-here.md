## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Two-syscall IPC framing | `send_ipc_message` wrote JSON then `\n` in two separate `write_all` calls, allowing a race where the reader processes the JSON between syscalls, closes the stream, and the `\n` write hits EPIPE | blocking | Always construct the full newline-delimited frame in memory first, then single `write_all`. See `pam-prmana/src/sudo.rs:877` — fixed in commit 047532d. Same rule for any future newline-framed socket protocol. |
| "Privilege separation" claim without helper | ADR-021 labels helper split as "Decision" but admits v1.0 may ship without it — claims security properties the shipped code doesn't have | blocking | README + security docs must not use phrase "privilege separation" for SCIM until the helper actually exists and passes distro matrix testing. Use "perimeter hardening" instead. |
| cargo test --all-features on cold runner | 45+ min build time, hardware-gated tests ignored anyway, no feedback if it hangs | advisory | Split into `cargo test --workspace --no-run` (compile) + `cargo test --workspace -- --test-threads=4` (run, with timeout-minutes: 15) + `cargo check --workspace --all-features` (compile check). Hardware tests belong in integration-docker.yml. |
| GHCR images without login in CI | SPIRE and swtpm images are on ghcr.io which rate-limits anonymous pulls | advisory | Add `docker/login-action@v3` with `${{ secrets.GITHUB_TOKEN }}` before any compose that pulls from ghcr.io. |
| Docker bind-mount volume to non-existent path | SPIRE volume config bind-mounts /tmp/spire-agent/public which must exist before compose up | advisory | Always `mkdir -p` the bind-mount target in a pre-step. |
