After this plan, `sudo` commands with `step_up_method = "push"` or `"fido2"` trigger a CIBA push notification to the user's phone or FIDO2 authenticator.

Output: Working CIBA step-up flow from PAM sudo -> agent IPC -> IdP CIBA -> user approval -> PAM success/failure.

**Step D: Implement handle_step_up()** (`unix-oidc-agent/src/daemon/socket.rs`):

1. Guard: check no pending step-up exists for this username (iterate `pending_step_ups` values checking `username`). If one is active AND not finished, return `AgentResponse::error("Step-up already in progress for this user", "STEP_UP_IN_PROGRESS")`.
2. Read `oidc_issuer` from `AgentState` — if `None`, return error "Agent not logged in or OIDC config missing".
3. Read `oidc_client_id` from `AgentState` — if `None`, return error.
4. Read `oidc_client_secret` from `AgentState` via `.as_ref().map(|s| s.expose_secret().to_string())` — this is an audit boundary for CIBA HTTP calls.
5. Fetch OIDC discovery: async GET to `{issuer}/.well-known/openid-configuration`, parse as `OidcDiscovery` (import from `pam_unix_oidc::oidc::OidcDiscovery`). Use a reqwest::Client with 10s timeout.
6. Construct `CibaClient::new(&discovery, &client_id, client_secret.as_deref())` — propagate errors.
7. Build `binding_message` via `build_binding_message(&command, &hostname)`.
8. Determine `acr_values`: if `method == "fido2"` then `Some(ACR_PHR)`, else `None`.
9. Determine `login_hint`: use the `username` parameter from the IPC request. This is the Unix username. Note: if the IdP expects email rather than username as `login_hint`, the operator must configure their IdP to accept username-based login_hint. Log a debug message: "Using Unix username as CIBA login_hint" (see RESEARCH.md Open Question #1 — login_hint_claim config is deferred to a future enhancement).
10. POST to `ciba_client.backchannel_endpoint()` with `ciba_client.build_backchannel_auth_params(login_hint, &binding_message, acr_values)`. Parse 200 response as `BackchannelAuthResponse`. On error, return `AgentResponse::error(...)`.
11. Generate `correlation_id` via `uuid::Uuid::new_v4().to_string()`.
12. Build owned token poll params from `ciba_client.build_ciba_token_params(&auth_response.auth_req_id)` — convert `(&str, &str)` pairs to `(String, String)` for the spawned task.
13. Determine `acr_required`: if method == "fido2" then `Some(ACR_PHR.to_string())`, else `None`.
14. Spawn `poll_ciba()` as Tokio task with: http_client, token_endpoint (owned String), owned params, interval Duration, timeout Duration, acr_required.
15. Store `PendingStepUp { handle, username, expires_at }` in `pending_step_ups` keyed by `correlation_id`.
16. Return `AgentResponse::step_up_pending(correlation_id, auth_response.expires_in, auth_response.interval)`.

**Step E: Implement poll_ciba() async function** (`unix-oidc-agent/src/daemon/socket.rs`):

```rust
async fn poll_ciba(
    http: reqwest::Client,
    token_endpoint: String,
    params: Vec<(String, String)>,
    mut interval: Duration,
    timeout: Duration,
    acr_required: Option<String>,
) -> StepUpOutcome
```
- Wrap the loop in `tokio::time::timeout(timeout, async { ... })`.
- Loop: `tokio::time::sleep(interval).await`, then POST `token_endpoint` with `params` as form body.
- On 200 OK: parse as `CibaTokenResponse`. If `acr_required` is `Some(ref required)`, extract `acr` claim from `id_token` (decode JWT middle segment, parse as JSON, read `acr` field). Call `validate_acr(required, acr.as_deref())`. On ACR failure, return `TimedOut { reason: "acr_failed", user_message: "Step-up authentication did not meet required assurance level" }`. On success, return `Complete { acr, session_id: uuid::Uuid::new_v4().to_string() }`.
- On non-200: parse body as JSON with `error` field. Use `parse_ciba_error(error)` to match:
  - `AuthorizationPending` -> continue
  - `SlowDown` -> `interval += Duration::from_secs(5)`, continue
  - `AccessDenied` -> return `TimedOut("denied", "Step-up request was denied")`
  - `ExpiredToken` -> return `TimedOut("expired", "Approval window expired")`
  - other -> return `TimedOut("error", format!("CIBA error: {}", error_desc))`
- On timeout: return `TimedOut("timeout", "Step-up approval timed out")`
