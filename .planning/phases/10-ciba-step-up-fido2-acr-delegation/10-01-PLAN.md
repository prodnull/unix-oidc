    - CibaClient::new validates backchannel_authentication_endpoint is present in discovery, and poll mode is supported; returns NoCibaEndpoint or DeliveryModeNotSupported errors
  </behavior>
  <action>
Create `pam-unix-oidc/src/ciba/client.rs`:

1. `pub fn build_binding_message(command: &str, hostname: &str) -> String`:
   - Extract executable basename: split command on whitespace, take first token, split on '/' take last segment. Fallback to "unknown" if empty.
   - Format as `"sudo {exe} on {hostname}"`
   - If result exceeds 64 chars, truncate to 64 chars (CIBA spec recommends short binding_message for authenticator UI)
   - Security: Do NOT include command arguments — they may contain sensitive paths/data (Pitfall 6 from research)

2. `pub struct CibaClient` with fields:
   - `backchannel_endpoint: String`
   - `token_endpoint: String`
   - `client_id: String`
   - `client_secret: Option<String>`

3. `impl CibaClient`:
   - `pub fn new(discovery: &OidcDiscovery, client_id: &str, client_secret: Option<&str>) -> Result<Self, CibaError>`:
     - Check `discovery.backchannel_authentication_endpoint.is_some()` — return `CibaError::NoCibaEndpoint` if None
     - Check `discovery.backchannel_token_delivery_modes_supported` contains "poll" — return `CibaError::DeliveryModeNotSupported` if poll not present. If the field is None, assume poll is supported (some IdPs omit the field when only poll is available)
     - Store endpoints and credentials

   - `pub fn build_backchannel_auth_params<'a>(&'a self, login_hint: &'a str, binding_message: &'a str, acr_values: Option<&'a str>) -> Vec<(&'static str, &'a str)>`:
     - Build form params per CIBA Core 1.0 section 7.1
     - Always include: client_id, scope=openid, login_hint, binding_message
     - Conditionally include: client_secret (if set), acr_values (if Some)

   - `pub fn build_ciba_token_params<'a>(&'a self, auth_req_id: &'a str) -> Vec<(&'static str, &'a str)>`:
     - Build form params per CIBA Core 1.0 section 10.1
     - Use `CIBA_GRANT_TYPE` constant
     - Include: grant_type, client_id, auth_req_id
     - Conditionally include: client_secret (if set)

   - Accessor: `pub fn token_endpoint(&self) -> &str`
   - Accessor: `pub fn backchannel_endpoint(&self) -> &str`

Note: The actual HTTP calls will be made by the agent daemon (Plan 03), not by CibaClient directly. CibaClient only builds request parameters and provides endpoint URLs. This keeps the PAM crate free of async dependencies for CIBA.
