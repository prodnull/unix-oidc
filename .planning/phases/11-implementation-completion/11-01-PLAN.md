<task type="auto">
  <name>Task 2: Create DPoP binding E2E test script</name>
  <files>test/tests/test_dpop_binding.sh</files>
  <action>
Create `test/tests/test_dpop_binding.sh` -- a focused ~60-line shell script that validates Keycloak issues DPoP-bound tokens with correct `cnf.jkt`. Use the same helper patterns from `test_token_exchange.sh` (openssl EC key generation, base64url encoding, JWK construction, DPoP proof signing).

The script must:

1. **Check prerequisites**: `curl`, `jq`, `openssl`, `xxd` must be installed (exit with clear error if not).

2. **Generate a P-256 EC keypair** using `openssl ecparam -name prime256v1 -genkey -noout`.

3. **Extract JWK coordinates** (x, y) from the public key. Use `openssl ec -pubout -outform DER` and extract the raw x/y coordinates from the uncompressed point (byte 0x04 followed by 32 bytes x, 32 bytes y). Base64url-encode both.

4. **Compute the JWK thumbprint** per RFC 7638: SHA-256 of the canonical JSON `{"crv":"P-256","kty":"EC","x":"...","y":"..."}` (members in lexicographic order). Base64url-encode the hash.

5. **Construct a DPoP proof JWT**: Header `{"typ":"dpop+jwt","alg":"ES256","jwk":{...}}`, payload `{"jti":"<uuid>","htm":"POST","htu":"<token_endpoint>","iat":<now>}`. Sign with ES256 using the generated key.

6. **Request a token** from `http://localhost:8080/realms/unix-oidc-test/protocol/openid-connect/token` with:
   - `grant_type=password` (the realm has `directAccessGrantsEnabled: true`)
   - `client_id=unix-oidc` + `client_secret=test-secret`
   - `username=testuser`, `password=testpass`
   - `DPoP: <proof_jwt>` header

7. **Validate the response**:
   - Token request returns 200
   - Decode the access token (base64url decode the payload part)
   - Assert `cnf.jkt` field exists in the token payload
   - Assert `cnf.jkt` value matches the computed JWK thumbprint from step 4
   - Print PASS/FAIL with the thumbprint values for debugging

8. **Negative test**: Attempt the same token request WITHOUT the DPoP header. Assert Keycloak returns 400 (when `dpop.bound.access.tokens: true`, Keycloak rejects requests without DPoP proof).

Configuration via env vars (with defaults):
- `KEYCLOAK_URL` (default: `http://localhost:8080`)
- `REALM` (default: `unix-oidc-test`)
- `CLIENT_ID` (default: `unix-oidc`)
- `CLIENT_SECRET` (default: `test-secret`)
- `TEST_USERNAME` (default: `testuser`)
- `TEST_PASSWORD` (default: `testpass`)

Use the exact same base64url encoding function from `test_token_exchange.sh`: `base64 | tr '+/' '-_' | tr -d '='`.

The script must `set -e` and clean up temp files on EXIT via a trap.
  </action>
