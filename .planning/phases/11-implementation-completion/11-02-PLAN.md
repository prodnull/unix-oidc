Create `unix-oidc-agent/tests/daemon_lifecycle.rs` as a Rust integration test.

**CRITICAL**: The `Shutdown` handler in `socket.rs` calls `std::process::exit(0)` immediately WITHOUT sending a response back on the socket. The test must:
1. Send the shutdown JSON command
2. NOT attempt to read a response (it will hang or get EOF)
3. Wait for the child process to exit via `child.wait()`

Structure:
```rust
//! Agent daemon lifecycle integration test (TEST-04).
//!
//! Starts the unix-oidc-agent binary, sends IPC commands, validates responses.
//! Run: cargo test -p unix-oidc-agent --test daemon_lifecycle

use std::io::{BufRead, BufReader, Write};
use std::os::unix::net::UnixStream;
use std::path::PathBuf;
use std::process::{Command, Stdio};
use std::time::{Duration, Instant};
```

Implementation details:

1. **Socket path**: Use `std::env::temp_dir().join(format!("unix-oidc-test-{}.sock", std::process::id()))` for uniqueness. Clean up on exit.

2. **Start daemon**: `Command::new(env!("CARGO_BIN_EXE_unix-oidc-agent")).args(["serve", "--socket", socket_str]).stdout(Stdio::piped()).stderr(Stdio::null()).spawn()`. The `CARGO_BIN_EXE_unix-oidc-agent` env var is set automatically by cargo for integration tests.

3. **Wait for socket**: Poll `socket_path.exists()` with 50ms intervals up to 2 seconds. Assert it appears.

4. **Status command**: Connect `UnixStream::connect(&socket)`, write `{"action":"status"}\n`, read one line, parse as JSON. Assert `status == "success"`. Assert `data.is_logged_in == false` (fresh daemon, no credentials).

5. **GetProof command**: New connection (IPC is one-request-per-connection). Write `{"action":"get_proof","target":"test.example.com","method":"SSH"}\n`, read response. Assert `status == "error"` (unauthenticated daemon cannot generate proofs).

6. **Shutdown**: New connection. Write `{"action":"shutdown"}\n`. Do NOT read response. Drop the stream. Call `child.wait()` with a timeout (use `child.try_wait()` in a loop, up to 5 seconds). Assert process exited.

7. **Cleanup**: Remove socket file if still present. Use a Drop guard or explicit cleanup.

Add `serde_json` as a dev-dependency in `unix-oidc-agent/Cargo.toml` if not already present (check first -- it likely is since the agent uses it in production code).

The test function should be `#[test] fn test_daemon_lifecycle()` (not async -- it uses blocking std::os::unix::net::UnixStream and std::process::Command).
