       - `pub fn secure_remove(path: &Path) -> Result<(), SecureDeleteError>`:
         - If !path.exists(), return Err(NotFound)
         - Get file size from metadata
         - If size == 0, just unlink and return
         - Open file for writing (don't truncate -- use OpenOptions::write(true))
         - Three-pass overwrite (DoD 5220.22-M style per user decision):
           - Pass 1: fill buffer with random bytes, write over entire file, fsync
           - Pass 2: complement (XOR 0xFF) the random buffer, write over entire file, fsync
           - Pass 3: fill buffer with new random bytes, write over entire file, fsync
         - Use a reasonable buffer size (e.g., 4096 bytes) for large files
         - If any overwrite pass fails: log WARN, continue to unlink (best-effort per user decision)
         - fsync after each pass via file.sync_all()
         - unlink via std::fs::remove_file
       - `pub fn detect_cow_filesystem(path: &Path) -> bool`:
         - `#[cfg(target_os = "linux")]`: check if path is on btrfs by reading /proc/mounts or using statfs and checking f_type for BTRFS_SUPER_MAGIC (0x9123683E)
         - `#[cfg(target_os = "macos")]`: check if filesystem is APFS (statfs, f_fstypename == "apfs")
         - `#[cfg(not(any(target_os = "linux", target_os = "macos")))]`: return false
       - `pub fn detect_rotational_device(path: &Path) -> Option<bool>`:
         - `#[cfg(target_os = "linux")]`: resolve path to device, check /sys/block/{dev}/queue/rotational (0 = SSD, 1 = HDD)
         - `#[cfg(not(target_os = "linux"))]`: return None (unknown)
       - `pub fn log_storage_advisories(storage_dir: &Path)`:
         - Call detect_cow_filesystem, detect_rotational_device
         - If CoW: `warn!("Storage directory is on a CoW filesystem (btrfs/APFS). Secure delete overwrites may not erase all copies. Full-disk encryption is recommended.")`
