  <action>
1. Create `pam-unix-oidc/tests/multi_idp_integration.rs` as an integration test file (outside src/).

2. Use `#[cfg(feature = "test-mode")]` to gate the entire file — these tests require test-mode for JWT construction without real JWKS.

3. Create helper functions:
   - `make_test_token(iss: &str, sub: &str, username: &str) -> String` — constructs a minimal JWT with the given claims. Base64-encode a JSON header (`{"alg":"ES256","typ":"JWT"}`) and payload, append dummy signature. This works with test-mode's `new_insecure_for_testing()`.
   - `make_multi_issuer_policy(issuers: Vec<IssuerConfig>) -> PolicyConfig` — builds a PolicyConfig with the given issuers array and sensible defaults for other fields.
   - `make_issuer_config(url: &str, dpop: EnforcementMode, transforms: Vec<TransformConfig>) -> IssuerConfig` — shorthand for building test IssuerConfig.

4. Write integration tests:

   **MIDP-01: Config loading**
   - `test_two_issuer_policy_loads_from_yaml`: Load `test/fixtures/policy/policy-multi-idp.yaml` via `PolicyConfig::load_from()`, assert `issuers.len() == 2`, check each issuer's URL.
   - `test_effective_issuers_backward_compat`: Set `OIDC_ISSUER` env var, create policy with empty issuers, call `effective_issuers()`, verify single-element result.

   **MIDP-02: Per-issuer DPoP enforcement**
   - `test_dpop_strict_rejects_bearer_only`: Create token for strict-DPoP issuer, call `authenticate_multi_issuer()` with `dpop_proof: None`, verify error.
   - `test_dpop_disabled_accepts_bearer`: Create token for disabled-DPoP issuer, call `authenticate_multi_issuer()` with `dpop_proof: None`, verify success.

   **MIDP-03: Per-issuer claim mapping**
   - `test_strip_domain_per_issuer`: Issuer A has `strip_domain` transform, token has `preferred_username: "alice@corp.example"`, verify AuthResult.username == "alice". Issuer B has no transform, verify AuthResult.username == "alice@corp.example".

   **MIDP-04: ACR mapping**
   - `test_acr_mapping_translates_values`: Create policy with Keycloak-style ACR mapping. Verify that `AcrMappingConfig::mappings` translates `"urn:keycloak:acr:loa2"` to the canonical value. (Unit-level — ACR mapping is applied post-auth, verify the config deserializes and lookups work.)

   **MIDP-05: Group mapping**
   - `test_group_mapping_defaults_to_nss`: Create IssuerConfig without group_mapping, verify it defaults to NssOnly.
   - `test_group_mapping_token_claim_mode`: Create IssuerConfig with `source: TokenClaim`, verify `claim` field.

   **MIDP-06: Issuer routing**
   - `test_known_issuer_routes_correctly`: Token with `iss` matching config → success.
   - `test_unknown_issuer_rejected`: Token with `iss` not in config → `AuthError::UnknownIssuer`.
   - `test_issuer_routing_normalizes_trailing_slash`: Token `iss` without slash, config with slash → still matches.

   **MIDP-07: JWKS independence + JTI scoping**
   - `test_jwks_registry_independent_per_issuer`: Get provider for A and B, verify `!Arc::ptr_eq`.
   - `test_jti_same_value_different_issuers_no_collision`: Authenticate token with JTI "abc" from issuer A, then token with JTI "abc" from issuer B — both succeed.
   - `test_jti_same_value_same_issuer_rejected`: Authenticate token with JTI "abc" from issuer A twice — second rejected.

   **MIDP-08: Graceful defaults**
   - `test_issuer_without_optional_fields_loads`: IssuerConfig with only `issuer_url` and `client_id` deserializes successfully, dpop_enforcement defaults to Strict.

   **Adversarial**
   - `test_duplicate_issuer_urls_rejected`: Policy with two identical issuer_url values → load error.

5. Update `test/fixtures/policy/policy-multi-idp.yaml` if needed (may already exist from Plan 01 — enhance if needed to cover ACR mapping and group mapping examples).

Anti-patterns to avoid:
- Do NOT use real network calls. All tests use test-mode (no JWKS fetch needed for signature bypass).
- Do NOT rely on global state between tests for JTI collision tests. Create fresh JtiCache instances or use unique JTI values with issuer-scoped keys.
