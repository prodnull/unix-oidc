<objective>
Refactor DPoP proof generation to support external signers and implement YubiKeySigner via PKCS#11.

Purpose: The current `generate_dpop_proof()` takes `&SigningKey` directly, which hardware signers cannot expose. This plan splits JWT construction from signing, then implements the YubiKey backend that uses cryptoki to sign on the device. This is the riskiest piece (PKCS#11 integration) and acts as the spike for the CKM_ECDSA_SHA256 path.

Output: Refactored dpop.rs with `build_dpop_message`/`assemble_dpop_proof`, YubiKeySigner behind `--features yubikey` with both `open()` (read existing key) and `provision()` (generate new key) constructors, PinCache with SecretString timeout.
</objective>

**Step 1 — Refactor dpop.rs:** Factor `generate_dpop_proof()` into three functions:

1. `build_dpop_message(public_key_jwk: &serde_json::Value, method: &str, target: &str, nonce: Option<&str>) -> Result<String, DPoPError>` — builds the `header_b64.claims_b64` string. The header embeds the JWK. Returns the unsigned message string.

2. `assemble_dpop_proof(message: &str, sig_rs_bytes: &[u8]) -> Result<String, DPoPError>` — base64url-encodes the raw r||s signature bytes (must be exactly 64 bytes for P-256) and appends to message: `message.sig_b64`. Return the complete JWT string. Add a length check: if `sig_rs_bytes.len() != 64`, return a new `DPoPError::InvalidSignatureLength` variant.

3. Keep `generate_dpop_proof()` as a convenience that calls `build_dpop_message` + p256 sign + `assemble_dpop_proof`. SoftwareSigner continues to call `generate_dpop_proof()` unchanged.

Add `InvalidSignatureLength` variant to `DPoPError`. Add `HardwareSigner(String)` variant to `DPoPError` for hardware backend errors.

**YubiKeySigner struct:**
```rust
pub struct YubiKeySigner {
    pkcs11_path: String,
    slot_id: u8,          // YKCS11 key ID: 9a=0x01, 9c=0x02, 9d=0x03, 9e=0x04
    pin_cache: PinCache,
    thumbprint: String,
    public_key_jwk: serde_json::Value,
}
```

**Constructor `YubiKeySigner::open(slot_spec: &str, config: &SignerConfig) -> anyhow::Result<Self>`:**
1. Parse slot from spec (e.g., "9a" -> key_id 0x01). Map: "9a"->1, "9c"->2, "9d"->3, "9e"->4. Error if unknown slot.
2. Determine PKCS#11 library path: config override > platform default (`/usr/lib/libykcs11.so` Linux, `/usr/local/lib/libykcs11.dylib` macOS).
3. Open a temporary PKCS#11 session to read the public key:
   - `Pkcs11::new(&pkcs11_path)?` + `initialize(CInitializeArgs::OsThreads)?`
   - Get first slot with token: `get_slots_with_token()?`
   - Open RO session (no login needed for public key read)
   - Find object: `ObjectClass::PUBLIC_KEY` + `KeyType::EC` + `Attribute::Id(vec![key_id])`
   - Read `AttributeType::EcPoint` — this returns DER OCTET STRING wrapping the uncompressed EC point. Strip DER wrapper (first 2 bytes are TLV: 0x04 0x41 for 65-byte P-256 uncompressed). The remaining 65 bytes start with 0x04 prefix followed by 32-byte x and 32-byte y.
   - If no key found or not P-256, return error: "No P-256 key found in YubiKey PIV slot {slot}. Use `unix-oidc-agent provision --signer yubikey:{slot}` to generate one."
4. Compute JWK and thumbprint from x,y coordinates using `compute_ec_thumbprint` logic (build JWK JSON, SHA-256 hash canonical form). Since we have raw x,y bytes (not a VerifyingKey), compute thumbprint inline: base64url encode x and y, build canonical JSON `{"crv":"P-256","kty":"EC","x":"...","y":"..."}`, SHA-256 hash, base64url encode.
5. Session drops (PCSC released).
6. Build `PinCache::new(config.yubikey.pin_cache_timeout.unwrap_or(28800))`.

**Provisioning `YubiKeySigner::provision(slot_spec: &str, config: &SignerConfig) -> anyhow::Result<Self>`:**
This method generates a new P-256 key pair on the YubiKey via PKCS#11 `C_GenerateKeyPair`. Plan 03-03's `provision_signer()` delegates to this method for the yubikey case.

1. Parse slot from spec (same mapping as `open()`).
2. Determine PKCS#11 library path (same logic as `open()`).
3. Open a PKCS#11 session:
   - `Pkcs11::new(&pkcs11_path)?` + `initialize(CInitializeArgs::OsThreads)?`
   - `get_slots_with_token()?`, get first slot
   - `open_rw_session(slot)?`
4. Login with PIN: get PIN via `PinCache::new(0).get_or_prompt("YubiKey PIN (required for key generation): ")?` (timeout=0, always prompt for provisioning).
   - `session.login(UserType::User, Some(&AuthPin::new(pin_str.into())))?`
   - Handle `CKR_PIN_INCORRECT` and `CKR_PIN_LOCKED` same as in `sign_proof()`.
5. First, check if slot already has a P-256 key. Find `ObjectClass::PUBLIC_KEY` + `KeyType::EC` + `Attribute::Id(vec![key_id])`. If found:
   - Read `EcPoint`, extract x,y, verify it is P-256 (65-byte uncompressed point).
   - If compatible: log info "Existing P-256 key found in slot {slot}, adopting it." Build JWK + thumbprint from x,y, skip generation.
   - If incompatible (not P-256): return error "YubiKey PIV slot {slot} contains a non-P-256 key. To overwrite, first clear the slot with `ykman piv keys delete {slot}`, then re-run provision."
6. If no existing key, generate via `C_GenerateKeyPair`:
   - P-256 curve OID: `&[0x06, 0x08, 0x2a, 0x86, 0x48, 0xce, 0x3d, 0x03, 0x01, 0x07]` (DER-encoded OID 1.2.840.10045.3.1.7)
   - Public key template: `[CKA_EC_PARAMS = p256_oid, CKA_TOKEN = true, CKA_ID = vec![key_id], CKA_VERIFY = true]`
   - Private key template: `[CKA_TOKEN = true, CKA_PRIVATE = true, CKA_SENSITIVE = true, CKA_EXTRACTABLE = false, CKA_SIGN = true, CKA_ID = vec![key_id]]`
   - Call `session.generate_key_pair(&Mechanism::EcKeyPairGen, &pub_template, &priv_template)?`
   - On success: read `EcPoint` from the returned public key handle, extract x,y.
7. Build JWK and thumbprint from x,y (same as `open()`).
8. Session drops (PCSC released).
9. Build `PinCache::new(config.yubikey.pin_cache_timeout.unwrap_or(28800))`.
10. Return `Self` with populated fields.

**DPoPSigner implementation:**
- `thumbprint()` -> returns stored thumbprint
- `public_key_jwk()` -> returns stored JWK
- `sign_proof()`:
  1. Call `build_dpop_message(&self.public_key_jwk, method, target, nonce)?`
  2. Open new PKCS#11 session (strict open-sign-close per HW-04):
     - `Pkcs11::new(&self.pkcs11_path)?` + `initialize(CInitializeArgs::OsThreads)?`
     - `get_slots_with_token()?`, get slot
     - `open_rw_session(slot)?`
  3. Login: `session.login(UserType::User, Some(&AuthPin::new(pin_str.into())))?`
     - Get PIN via `self.pin_cache.get_or_prompt("YubiKey PIN: ")?`
     - On `CKR_PIN_INCORRECT` (map from cryptoki error): call `self.pin_cache.clear()`, return error "YubiKey PIN incorrect. PIN cache cleared; next attempt will re-prompt."
     - On `CKR_PIN_LOCKED`: return error "YubiKey PIN is locked after too many failed attempts. Use `ykman piv access unblock-pin` to unblock with your PUK."
  4. Find private key: `ObjectClass::PRIVATE_KEY` + `KeyType::EC` + `Attribute::Id(vec![key_id])`
  5. Try `session.sign(&Mechanism::EcdsaSha256, key_handle, message.as_bytes())?`
     - If `CKR_MECHANISM_INVALID`: fall back to prehash path — `sha2::Sha256::digest(message.as_bytes())`, then `session.sign(&Mechanism::Ecdsa, key_handle, &digest)?`
  6. Session drops (PCSC released — HW-04 satisfied).
  7. Call `assemble_dpop_proof(&message, &sig_bytes)?`
