**Step 1 — Add `build_signer` factory to `hardware/mod.rs`:**

```rust
pub fn build_signer(signer_spec: &str, config: &SignerConfig) -> anyhow::Result<Arc<dyn DPoPSigner>> {
    if signer_spec == "software" || signer_spec.is_empty() {
        // SoftwareSigner is always available — handled by caller (load_or_create_signer)
        anyhow::bail!("Software signer should be handled by caller");
    }

    #[cfg(feature = "yubikey")]
    if let Some(slot) = signer_spec.strip_prefix("yubikey:") {
        return Ok(Arc::new(crate::crypto::YubiKeySigner::open(slot, config)?));
    }

    #[cfg(not(feature = "yubikey"))]
    if signer_spec.starts_with("yubikey:") {
        anyhow::bail!("YubiKey support not compiled in. Rebuild with `--features yubikey`.");
    }

    #[cfg(feature = "tpm")]
    if signer_spec == "tpm" {
        return Ok(Arc::new(crate::crypto::TpmSigner::load(config)?));
    }

    #[cfg(not(feature = "tpm"))]
    if signer_spec == "tpm" {
        anyhow::bail!("TPM support not compiled in. Rebuild with `--features tpm`.");
    }

    anyhow::bail!("Unknown signer: '{}'. Valid options: software, yubikey:<slot>, tpm", signer_spec)
}
```

Add `provision_signer(signer_spec: &str, config: &SignerConfig) -> anyhow::Result<(String, serde_json::Value, String)>` that returns (signer_type_string, jwk, thumbprint):
- For `yubikey:<slot>`: call YubiKeySigner provisioning (generate P-256 key in PIV slot). Per CONTEXT.md, this uses PKCS#11 `C_GenerateKeyPair` with `CKM_EC_KEY_PAIR_GEN` and P-256 curve OID. If slot already has a compatible P-256 key, adopt it (read public key, return). If incompatible, error with guidance. Returns `("yubikey:<slot>", jwk, thumbprint)`.
- For `tpm`: call `TpmSigner::provision(config)?`. Returns `("tpm", jwk, thumbprint)`.
- For `software`: bail — software signer auto-generates at login, no provisioning needed.
- Errors clearly when hardware not found per CONTEXT.md decisions.

Create `docs/hardware-key-setup.md` covering:

**1. Overview:**
- unix-oidc-agent supports three signer backends: software (default), YubiKey (PKCS#11), TPM 2.0
- Hardware signers store the DPoP private key on the device — the key never leaves the hardware
- Feature flags: `--features yubikey`, `--features tpm` — not included in default build

**2. YubiKey PIV Setup:**
- Prerequisites: YubiKey 5 series (or any PIV-capable YubiKey), `pcscd` running (`sudo systemctl start pcscd`), `libykcs11` installed
- Install on Ubuntu/Debian: `sudo apt install pcscd libykcs11 yubico-piv-tool`
- Install on macOS: `brew install yubico-piv-tool` (pcscd is built-in as `com.apple.ifdreader`)
- Supported PIV slots: 9a (Authentication - recommended), 9c (Digital Signature), 9d (Key Management), 9e (Card Authentication)
- Provisioning: `unix-oidc-agent provision --signer yubikey:9a`
  - Generates P-256 key in the specified slot
  - If slot already has a P-256 key, adopts it without regenerating
  - If slot has an incompatible key, errors with guidance
- Login: `unix-oidc-agent login --signer yubikey:9a --issuer https://your-idp.com`
- PIN management:
  - Default YubiKey PIV PIN: 123456 (change immediately: `ykman piv access change-pin`)
  - Default PUK: 12345678
  - PIN is cached for 8 hours by default (configurable in `~/.config/unix-oidc/signer.yaml`)
  - After 3 wrong PINs, the YubiKey locks. Unblock with PUK: `ykman piv access unblock-pin`
  - After 3 wrong PUKs, the PIV application is permanently locked. Factory reset: `ykman piv reset` (destroys all PIV keys)

**3. TPM 2.0 Setup:**
- Prerequisites: TPM 2.0 hardware (physical or vTPM), `tpm2-abrmd` resource manager running
- Install on Ubuntu/Debian: `sudo apt install tpm2-abrmd tpm2-tools`
- Start resource manager: `sudo systemctl enable --now tpm2-abrmd`
- Verify TPM available: `tpm2_getcap properties-fixed` (should show manufacturer info)
- Provisioning: `unix-oidc-agent provision --signer tpm`
  - Probes P-256 capability first (errors if TPM lacks NistP256)
  - Creates unrestricted signing key under Owner hierarchy
  - Stores key at persistent handle 0x81000001 (configurable)
- Login: `unix-oidc-agent login --signer tpm --issuer https://your-idp.com`
- Cloud vTPM notes:
  - AWS Nitro: vTPM available on Nitro instances; P-256 generally supported
  - GCP: Shielded VM vTPM available; verify P-256 with provision command
  - Azure: vTPM on Trusted Launch VMs; verify P-256 with provision command
  - Run `unix-oidc-agent provision --signer tpm` on the instance to verify before deploying

**4. Configuration (`~/.config/unix-oidc/signer.yaml`):**
```yaml
yubikey:
  pkcs11_library: /usr/lib/libykcs11.so  # /usr/local/lib/libykcs11.dylib on macOS
  pin_cache_timeout: 28800                # 8 hours in seconds, 0 = always prompt

tpm:
  tcti: tabrmd                            # tabrmd (default) | device | mssim
  persistent_handle: 0x81000001
  pin_cache_timeout: 28800
```

**5. Troubleshooting:**

| Symptom | Cause | Fix |
|---------|-------|-----|
| "No YubiKey detected (is pcscd running?)" | pcscd not running | `sudo systemctl start pcscd` |
| "No P-256 key found in YubiKey PIV slot 9a" | Slot empty or incompatible key | Run `provision --signer yubikey:9a` |
| "YubiKey PIN is locked" | 3 wrong PIN attempts | `ykman piv access unblock-pin` with PUK |
| "TPM not available" | tpm2-abrmd not running | `sudo systemctl start tpm2-abrmd` |
| "This TPM does not support P-256" | TPM lacks NistP256 | Use software signer on this device |
| "Hardware signer 'yubikey:9a' unavailable" (daemon restart) | YubiKey unplugged | Plug in YubiKey and re-login |
| PCSC exclusive lock errors | Another app holds YubiKey | Close gpg-agent or other PCSC apps |
| "YubiKey support not compiled in" | Built without feature flag | Rebuild with `cargo build --features yubikey` |

**6. Security Considerations:**
- Hardware keys are non-exportable — the DPoP private key never leaves the device
- YubiKey PIV touch policy can add physical presence confirmation (`ykman piv keys set-touch-policy`)
- TPM keys survive reboots but are bound to the specific TPM — laptop change requires re-provisioning
- No silent fallback to software signer — if hardware unavailable, authentication fails explicitly
- PIN is cached in memory using SecretString; cache expires after configured timeout

**Reference RFC 9449 (DPoP) and PKCS#11 v2.40 spec where applicable.**
