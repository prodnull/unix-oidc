Implement the agent-side session lifecycle: background token auto-refresh, SessionClosed IPC handler with token revocation, DPoP key zeroization, and credential cleanup.

Purpose: Auto-refresh prevents mid-session token expiry on long SSH sessions. Revocation + cleanup on session close ensures credentials don't persist beyond their useful lifetime.
Output: Agent daemon handles refresh scheduling, session teardown, and full credential lifecycle.
</objective>

    - Refresh task spawns on login and calculates sleep duration as (token_lifetime * threshold / 100)
    - Refresh task re-arms after successful refresh with new token expiry
    - Refresh task retries on failure: 5s, 10s, 20s, 40s backoff, max 3 retries
    - Refresh task sets refresh_failed=true in state after exhausting retries
    - AbortHandle stored in AgentState; calling abort() cancels the refresh task

3. Implement spawn_refresh_task():
   ```rust
   fn spawn_refresh_task(
       state: Arc<parking_lot::RwLock<AgentState>>,  // or whatever lock type is used
       token_expires: i64,
       threshold_percent: u8,
   ) -> tokio::task::AbortHandle
   ```
   - Calculate sleep duration: `lifetime = token_expires - now; sleep_secs = lifetime * threshold / 100`.
   - If sleep_secs <= 0, log WARN and proceed directly to refresh (token near/past expiry).
   - Sleep, then call perform_token_refresh() in a retry loop.
   - Retry with exponential backoff: delays = [5, 10, 20, 40] seconds, max 3 retries (4 total attempts including first).
   - On success: read new token_expires from state, re-arm the loop (sleep until new threshold).
   - On final failure: set state.refresh_failed = true, log WARN, task exits. Session continues until natural expiry.
   - Return AbortHandle (via handle.abort_handle() before dropping JoinHandle).

4. Wire spawn_refresh_task into the login flow:
   - After successful login (where access_token and token_expires are set in AgentState), call spawn_refresh_task().
   - Store the returned AbortHandle in state.refresh_task.
   - Read threshold_percent from config (add to agent config or use hardcoded 80 for now — CONTEXT.md says "configurable" but the config is in policy.yaml on the PAM side; agent can read its own config or use a sensible default).

5. Add tests:

1. Implement revoke_token_best_effort():
   - Async function that spawns a blocking task (tokio::task::spawn_blocking) for the HTTP call.
   - Read revocation_endpoint from KEY_TOKEN_METADATA JSON. If absent, log WARN("No revocation endpoint configured, skipping revocation") and return.
   - POST to revocation_endpoint: form body [("token", access_token), ("token_type_hint", "access_token")], Basic Auth with client_id/client_secret from metadata.
   - reqwest::blocking::Client with 5s timeout.
   - Log outcome: success at INFO, non-2xx at WARN, error at WARN.
   - This function must never panic or propagate errors — it's fire-and-forget.

2. Implement cleanup_session():
   - Async function called in background after SessionClosed ACK.
   - Steps in order:
     a. Cancel refresh task: if let Some(handle) = state.refresh_task.take() { handle.abort(); }
     b. Revoke token (best-effort): call revoke_token_best_effort() with the current access_token.
     c. Clear agent state: set access_token = None, token_expires = None, username = None, refresh_failed = false.
     d. Delete stored credentials: call storage.delete() for KEY_ACCESS_TOKEN, KEY_REFRESH_TOKEN, KEY_DPOP_PRIVATE, KEY_TOKEN_METADATA. Log WARN on individual failures but continue.
     e. Drop signer: set state.signer = None. This triggers ZeroizeOnDrop on ProtectedSigningKey when the last Arc reference is dropped.
   - Log the full cleanup sequence at INFO level with session_id for audit trail.

3. Wire SessionClosed handler in the main request dispatch (socket.rs):
   ```rust
   AgentRequest::SessionClosed { session_id } => {
       // ACK immediately — PAM must return quickly
       let response = AgentResponse::session_acknowledged();
       send_response(&mut stream, &response).await?;

       // Spawn cleanup in background — do NOT await
       let state_clone = Arc::clone(&self.state);
       tokio::spawn(async move {
           cleanup_session(state_clone, &session_id).await;
       });
       return Ok(());
   }
   ```
