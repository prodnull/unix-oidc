<objective>
Create the session record lifecycle infrastructure in the PAM module: config types, session record write/delete, audit events, and the open_session/close_session implementations with cross-fork session ID correlation.

Purpose: This is the foundation for bounded session lifetimes. Without session records, the agent has no way to track active sessions for introspection or revocation.
Output: Working pam_sm_open_session / pam_sm_close_session with tmpfs session records, new audit events, and config types for introspection and session settings.
</objective>

   - Also store token_jti, token_exp, and issuer as PAM env vars for open_session to consume:
     `UNIX_OIDC_TOKEN_JTI`, `UNIX_OIDC_TOKEN_EXP`, `UNIX_OIDC_ISSUER`.

2. Implement open_session():
   - Read UNIX_OIDC_SESSION_ID from pamh.getenv(). If absent, log WARN and return PAM_SUCCESS (session tracking is best-effort).
   - Read pam_user, source_ip (PAM_RHOST), token metadata from PAM env vars.
   - Get sshd_pid from std::process::id().
   - Load SessionConfig from PolicyConfig::from_env() (use defaults on error).
   - Call ensure_session_dir(&config.session.session_dir).
   - Construct SessionRecord and call write_session_record().
   - Emit AuditEvent::session_opened() and log it.
   - Return PamError::SUCCESS (errors in record writing are logged at WARN, never fatal).

3. Implement close_session():
   - Read UNIX_OIDC_SESSION_ID from pamh.getenv(). If absent, log WARN and return PAM_SUCCESS.
   - Load SessionConfig from PolicyConfig::from_env().
   - Call delete_session_record(). If record found, compute duration = now - session_start.
   - Emit AuditEvent::session_closed() with duration and log it.
   - Send SessionClosed IPC to agent daemon (best-effort, 2s timeout on Unix socket connect). Use the existing agent socket path pattern from the codebase. If agent is unreachable, log WARN and continue.
   - ALWAYS return PamError::SUCCESS — never block session teardown.

4. The IPC client call in close_session: write a minimal blocking Unix socket send of `{"action":"session_closed","session_id":"..."}` followed by reading the ACK. Use std::os::unix::net::UnixStream with 2s timeout. This is in the PAM module (no tokio), so blocking is fine. If connect or send fails, log WARN and return success.
