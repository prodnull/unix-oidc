    2. Add `#[instrument]` to the following functions in `unix-oidc-agent/src/crypto/dpop.rs`:
       - `generate_dpop_proof()`: `#[instrument(skip(signer), fields(htm, htu))]` — skip the signer (contains key material), capture HTTP method and target URI.
       - `build_dpop_message()`: `#[instrument(skip(access_token, nonce))]` — skip sensitive token data, capture htm and htu.
       - `assemble_dpop_proof()`: `#[instrument(skip_all)]` — skip all params (message contains claims, sig_rs_bytes is binary). Just records entry/exit timing via the span.
       - Do NOT instrument `extract_jwk_from_proof()` — it operates on untrusted input and adding spans there could leak proof content to logs.

    Note: `validate_token()`, `verify_dpop_proof()`, and `fetch_jwks()` live in pam-unix-oidc which runs in a separate process (the sshd PAM stack), not in the agent process. They are out of scope for agent-crate tracing.
