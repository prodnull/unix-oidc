  <files>unix-oidc-agent/src/daemon/socket.rs</files>
  <action>
    1. In the serve() select! loop, after listener.accept() succeeds:
       - Call get_peer_credentials(&stream) per RESEARCH.md Pattern 3.
       - Get daemon UID: `let daemon_uid = unsafe { libc::getuid() };`
       - If peer_uid != daemon_uid: log WARN with peer_uid and daemon_uid, drop stream, continue.
       - If get_peer_credentials returns Err: log WARN "Peer credential retrieval failed — rejecting (fail-closed)", drop stream, continue.
       - On success: log DEBUG with peer_pid if available.
    2. In handle_connection(), wrap the request read loop with tokio::time::timeout per RESEARCH.md Pattern 4:
       - Read ipc_idle_timeout_secs from config. The AgentState or AgentServer needs access to the timeout value. Options:
         (a) Pass the timeout Duration as a parameter to handle_connection, or
         (b) Store it in AgentState, or
         (c) Pass AgentConfig reference to handle_connection.
       - Use `tokio::time::timeout(idle_duration, read_next_request(...))` wrapping each iteration of the read loop.
       - On timeout (Err(Elapsed)): log DEBUG "IPC connection closed: idle timeout", break loop.
       - On Ok(Ok(None)): client closed cleanly, break.
       - On Ok(Err(e)): log error, break.
       - On Ok(Ok(Some(req))): handle request as before.
    3. Ensure the idle timeout is per-read (resets after each successful request), not per-connection total.
    4. Add a test: spawn a UnixListener, connect a client, send no data, verify the connection is closed after the timeout period (use a short timeout like 1s for the test).
