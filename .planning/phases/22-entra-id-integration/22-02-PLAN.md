  <action>
    Create `pam-unix-oidc/tests/entra_integration.rs` with the following tests, all annotated
    with `#[ignore = "Requires ENTRA_* env vars"]`:

    **Helper functions:**
    - `entra_env(name: &str) -> String` — reads env var or panics with descriptive message
    - `entra_issuer() -> String` — constructs `https://login.microsoftonline.com/{tenant}/v2.0`
    - `entra_token() -> String` — reads ENTRA_TOKEN env var
    - `entra_validation_config() -> ValidationConfig` — builds config with issuer, client_id,
      jti_enforcement=Warn, clock_skew=60s
    - `entra_jwks_provider() -> Arc<JwksProvider>` — creates JwksProvider for the Entra issuer
    - `entra_single_issuer_policy() -> PolicyConfig` — builds a PolicyConfig with one Entra
      issuer entry: dpop disabled, email claim with strip_domain+lowercase,
      allow_unsafe_identity_pipeline=true, jti_enforcement=warn

    **Positive tests (ENTR-02, ENTR-03):**

    1. `test_entra_discovery_returns_valid_jwks_uri` (async):
       - Fetch OIDC discovery from `https://login.microsoftonline.com/{tenant}/v2.0/.well-known/openid-configuration`
       - Assert response contains `jwks_uri` field
       - Fetch the JWKS URI, assert it contains at least 1 key
       - Assert at least one key has `kty: "RSA"` and `alg: "RS256"`

    2. `test_entra_rs256_token_validates` (async):
       - Create ValidationConfig and JwksProvider for the Entra issuer
       - Call `validator.validate(&token)` with the ENTRA_TOKEN
       - Assert Ok result
       - Assert claims.iss starts with `https://login.microsoftonline.com/`
       - Assert claims.preferred_username is Some (profile scope was requested)

    3. `test_entra_token_has_expected_claims`:
       - Validate token, extract claims
       - Assert `preferred_username` is present
       - Assert `email` is present (email scope requested)
       - Assert `sub` is present and non-empty

    **UPN mapping tests (ENTR-04):**

    4. `test_entra_upn_strip_domain_maps_to_bare_username`:
       - Create UsernameMapper with email claim + strip_domain + lowercase transforms
       - Validate token to get claims
       - Call mapper.map(&claims)
       - Assert result is Ok and contains no `@` character
       - Assert result is all lowercase

    5. `test_entra_raw_preferred_username_preserves_domain`:
       - Create UsernameMapper with preferred_username claim, no transforms
       - Validate token to get claims
       - Call mapper.map(&claims)
       - Assert result contains `@` (UPN format preserved)

    **Bearer-only auth (ENTR-05):**

    6. `test_entra_bearer_auth_succeeds_without_dpop`:
       - Build single-issuer Entra PolicyConfig (dpop disabled, allow_unsafe=true)
       - Call authenticate_multi_issuer with token, dpop_proof=None
       - Assert result is NOT Err(DPoPRequired)
       - Assert result is NOT Err(UnknownIssuer)
       - Result will be UserNotFound (no SSSD in test env) — that's the expected endpoint,
         confirming the auth pipeline completed past DPoP and collision-safety

    **Negative tests (adversarial coverage):**

    7. `test_entra_wrong_tenant_rejected`:
       - Build ValidationConfig with a fake tenant ID in the issuer URL
       - Validate the real token against wrong-tenant config
       - Assert Err (issuer mismatch or signature verification failure)

    8. `test_entra_tampered_token_rejected`:
       - Take the real token, modify one character in the payload section
       - Validate with correct config
       - Assert Err (signature verification failure)

    9. `test_entra_unknown_issuer_rejected_by_multi_issuer`:
       - Build PolicyConfig with a Keycloak-only issuer (not Entra)
       - Call authenticate_multi_issuer with the Entra token
       - Assert Err(UnknownIssuer)

    **Important implementation notes:**
    - Use `#[tokio::test]` for async tests (discovery, validation)
    - All tests `#[ignore]` — only run when env vars present
    - Do NOT use `--features test-mode` — these tests verify REAL signatures
    - Import from `pam_unix_oidc::oidc::{TokenValidator, ValidationConfig}`,
      `pam_unix_oidc::oidc::jwks::JwksProvider`,
      `pam_unix_oidc::identity::mapper::UsernameMapper`,
      `pam_unix_oidc::policy::config::*`,
      `pam_unix_oidc::auth::{authenticate_multi_issuer, AuthError, DPoPAuthConfig}`
    - For the discovery test, use reqwest directly to fetch the JSON
    - Use `#![allow(clippy::unwrap_used, clippy::expect_used)]` at crate level (test file)
