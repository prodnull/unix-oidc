<objective>
Add `expected_audience` and `allow_unsafe_identity_pipeline` fields to IssuerConfig, wire
audience override into the auth path, create the Entra policy fixture, and produce the
Entra app registration setup guide.

Purpose: Entra access tokens use custom audience URIs (api://) and UPN-based username
claims with strip_domain transforms. The existing collision-safety gatekeeper hard-fails
strip_domain unconditionally, which blocks Entra's required transform chain. This plan
resolves both issues and delivers the ENTR-01 documentation deliverable.

Output: Modified config/auth/collision code, Entra policy fixture, setup guide.
