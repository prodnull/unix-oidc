    - Test: sanitize_terminal_output("https://evil.com\x1b[2J\x1b[H") strips the CSI clear-screen and cursor-home sequences
    - Test: sanitize_terminal_output("https://evil.com\x1b]0;pwned\x07") strips OSC title-set sequence
    - Test: sanitize_terminal_output("https://evil.com\x1bPmalicious\x1b\\") strips DCS sequence
    - Test: sanitize_terminal_output("https://evil.com\x1b_malicious\x1b\\") strips APC sequence
    - Test: sanitize_terminal_output("https://evil.com\x00\x01\x02") strips control characters but preserves the URL
    - Test: sanitize_terminal_output("https://login.example.com/path?q=hello%20world") preserves percent-encoded URLs
    - Test: sanitize_terminal_output("https://login.example.com/idp/\u{00E9}") preserves valid Unicode (internationalized domain names)
    - Test: sanitize_terminal_output("") returns ""
    - Test: When sanitization strips anything, the function returns (sanitized_string, true); when nothing stripped, returns (sanitized_string, false)
    - Test: sanitize_terminal_output strips \x9b (8-bit CSI) as well as \x1b[ (7-bit CSI)

  <action>
1. Create `unix-oidc-agent/src/sanitize.rs` module with `pub fn sanitize_terminal_output(input: &str) -> (String, bool)`.

2. Implementation approach:
   - Strip all C0 control characters (0x00-0x1F except \t, \n, \r which are safe in some contexts but should also be stripped for URIs)
   - Strip C1 control characters (0x80-0x9F)
   - Strip ANSI escape sequences: CSI (ESC [ ... final byte), OSC (ESC ] ... ST), DCS (ESC P ... ST), APC (ESC _ ... ST), PM (ESC ^ ... ST), SOS (ESC X ... ST)
   - ST (String Terminator) is ESC \ or 0x9C
   - Preserve all valid Unicode (>= U+00A0) including internationalized domain names
   - Return tuple of (sanitized string, was_modified) so caller can log when sanitization occurred

3. Register the module in `unix-oidc-agent/src/lib.rs` or `main.rs` module declarations.

4. In `main.rs`, wrap all `verification_uri` and `verification_uri_complete` display points (around lines 787-820) with `sanitize_terminal_output()`. When `was_modified` is true, log at WARN level: "Sanitized terminal escape sequences from verification_uri — raw bytes removed: {hex_of_removed_bytes}".

5. Do NOT reject the URI — graceful degradation per user decision. Strip and display the safe version.

    - Test: When D-Bus session uses encryption (DH negotiated), all modes proceed without warning

  <action>
1. Add `reject_plain_dbus_sessions` config field. Since this is an agent-side config (not PAM policy), determine the appropriate config location. Options: environment variable `UNIX_OIDC_REJECT_PLAIN_DBUS` or a new agent config struct. Use environment variable approach for simplicity (matches existing `UNIX_OIDC_STORAGE_BACKEND` pattern), with values "strict", "warn", "disabled". Default: "warn".

2. In `router.rs`, after the Secret Service probe succeeds in `detect_auto()` (around line 438), add a D-Bus session encryption check:
   - The `keyring` crate's Secret Service backend uses `oo7` (or `dbus-secret-service`) under the hood. Check whether the session was opened with encryption by inspecting the session algorithm.
   - If the underlying library does not expose session encryption status directly, use the `oo7` crate's `Service::open_session()` which returns an `Algorithm` enum (Plain or Encrypted). Call this probe separately.
   - If direct inspection is not feasible with the current `keyring` crate abstraction, use a standalone `oo7` D-Bus call to `org.freedesktop.secrets` → `OpenSession` with algorithm `dh-ietf1024-sha256-aes128-cbc-pkcs7` and check if the server agrees or falls back to `plain`.

3. Based on the `reject_plain_dbus_sessions` setting:
   - `strict`: If session is plain, log error "D-Bus Secret Service session is unencrypted — rejecting backend per strict policy", skip Secret Service, fall through to keyutils/file.
   - `warn`: If session is plain, log warning "D-Bus Secret Service session is unencrypted — credentials transit D-Bus in plaintext. Set UNIX_OIDC_REJECT_PLAIN_DBUS=strict to reject, or use full-disk encryption." Continue using Secret Service.
   - `disabled`: Skip the check entirely.

4. Emit a structured audit event (via `tracing::info!(target: "unix_oidc_audit", ...)`) when a plain session is detected, regardless of enforcement mode. Include: event_type="DBUS_PLAIN_SESSION", backend="secret-service", enforcement="{mode}". This gives fleet-level SIEM visibility.

5. The check runs once at startup during `detect()`, not on every storage operation. This matches the user decision from CONTEXT.md.

6. For testing: The D-Bus check is platform-specific (Linux only) and requires a running D-Bus session. Gate the probe behind `#[cfg(target_os = "linux")]`. Unit tests should test the enforcement logic (given a plain/encrypted probe result, does the correct action happen?) by extracting the decision logic into a testable function separate from the actual D-Bus call.
