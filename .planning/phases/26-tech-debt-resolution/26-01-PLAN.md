<objective>
Wire ACR mapping enforcement from IssuerConfig into the multi-issuer auth path (DEBT-02) and make JWKS TTL and HTTP timeout configurable per-issuer (DEBT-05).

Purpose: Two IssuerConfig fields that exist but are not wired into the auth pipeline. ACR enforcement is a security property operators explicitly configure; leaving it hardcoded as `None` silently ignores their configuration. JWKS TTL/timeout hardcoded constants prevent operators from tuning cache behavior per-issuer.

Output: ACR mapping enforcement active in auth.rs; per-issuer JWKS cache TTL and HTTP timeout configurable via policy.yaml.
</objective>
