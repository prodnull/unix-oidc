<task type="auto">
  <name>Task 2: Add structured audit events at five agent event points</name>
  <files>unix-oidc-agent/src/daemon/socket.rs</files>
  <action>
    Add `tracing::info!(target: "unix_oidc_audit", ...)` calls at the following five event points in socket.rs. Per user decision in CONTEXT.md, every audit event MUST include: `event_type`, `session_id`, `username`, and `outcome`. The `timestamp` field is automatically added by the tracing-subscriber JSON layer (configured in main.rs via `fmt::layer().json()`) and MUST NOT be added as a manual field (doing so would create a duplicate). Use `target: "unix_oidc_audit"` (the first macro argument, NOT a named field) per RESEARCH.md Pitfall 1. These are emitted IN ADDITION TO existing operational logs (do not replace them).

    **Deriving session_id:** At GetProof time the session_id is not yet known (it is created by pam_sm_open_session after auth succeeds). Use the username + a request-scoped identifier (e.g., the agent's internal connection ID or a generated UUID) as the session_id for Event 1. For Event 2, read session_id from AgentState if available, otherwise use the username. For Events 3-5, session_id is directly available from the IPC message fields. Where session_id is genuinely unavailable, emit `session_id = "n/a"` with a comment explaining why — never omit the field entirely.

    **Event 1 — AGENT_AUTH (GetProof handler):**
    After successful proof generation, emit:
    ```rust
    tracing::info!(
        target: "unix_oidc_audit",
        event_type = "authentication",
        session_id = %session_id,
        username = %username,
        outcome = "success",
        signer_type = %signer_type,
        "AGENT_AUTH"
    );
    ```
    Also emit on auth failure paths:
    ```rust
    tracing::info!(
        target: "unix_oidc_audit",
        event_type = "authentication",
        session_id = "n/a",
        username = %username,
        outcome = "failure",
        reason = %error_reason,
        "AGENT_AUTH"
    );
    ```

    **Event 2 — AGENT_REFRESH (spawn_refresh_task success path):**
    After successful token refresh:
    ```rust
    tracing::info!(
        target: "unix_oidc_audit",
        event_type = "token_refresh",
        session_id = %session_id,
        username = %username,
        outcome = "success",
        expires_in = expires_in,
        "AGENT_REFRESH"
    );
    ```

    **Event 3 — AGENT_SESSION_CLOSED (SessionClosed handler):**
    After processing session close:
    ```rust
    tracing::info!(
        target: "unix_oidc_audit",
        event_type = "session_closed",
        session_id = %session_id,
        username = %username,
        outcome = "success",
        "AGENT_SESSION_CLOSED"
    );
    ```
    Note: username may need to be read from agent state.

    **Event 4 — AGENT_STEP_UP (StepUp handler, after spawning CIBA):**
    ```rust
    tracing::info!(
        target: "unix_oidc_audit",
        event_type = "step_up_initiated",
        session_id = %parent_session_id.as_deref().unwrap_or("n/a"),
        username = %username,
        outcome = "pending",
        method = %method,
        parent_session_id = ?parent_session_id,
        "AGENT_STEP_UP"
    );
    ```

    **Event 5 — AGENT_STEP_UP_COMPLETE / AGENT_STEP_UP_TIMED_OUT (handle_step_up_result):**
    On complete:
    ```rust
    tracing::info!(
        target: "unix_oidc_audit",
        event_type = "step_up_complete",
        session_id = %sudo_session_id,
        username = %username,
        outcome = "success",
        sudo_session_id = %sudo_session_id,
        parent_session_id = ?parent_session_id,
        acr = ?acr,
        "AGENT_STEP_UP_COMPLETE"
    );
    ```
    On timeout:
    ```rust
    tracing::info!(
        target: "unix_oidc_audit",
        event_type = "step_up_timed_out",
        session_id = %parent_session_id.as_deref().unwrap_or("n/a"),
        username = %username,
        outcome = "failure",
        reason = %reason,
        "AGENT_STEP_UP_TIMED_OUT"
    );
    ```

    NOTE on `unwrap_or`: The templates above use `parent_session_id.as_deref().unwrap_or("n/a")` for the `session_id` field in step-up events where the parent session ID is optional. Since `#![deny(clippy::unwrap_used)]` is NOT active in the agent crate (only in pam-unix-oidc), `unwrap_or` is safe here. If the agent also denies it, use a `match` or `map_or` instead.

    After adding all events, run `cargo clippy --workspace -- -D warnings` to catch any format issues.
