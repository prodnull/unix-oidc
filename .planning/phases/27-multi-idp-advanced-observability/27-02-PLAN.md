<objective>
Add structured audit events for key lifecycle operations (generation, loading, destruction) in the agent daemon.

Purpose: Security auditors need to trace the complete lifecycle of DPoP and PQC key material. Currently key operations only emit tracing spans, not structured audit events suitable for SIEM ingestion. This plan adds KEY_GENERATED, KEY_LOADED, and KEY_DESTROYED audit events at the actual lifecycle points.
Output: Three new audit event types emitted from ProtectedSigningKey, HybridPqcSigner, and the daemon's key loading path.
</objective>

  <name>Task 1: Key lifecycle audit events for DPoP keys</name>
  <files>unix-oidc-agent/src/crypto/protected_key.rs, unix-oidc-agent/src/daemon/socket.rs</files>
  <behavior>
    - Test 1: `ProtectedSigningKey::generate()` emits a tracing event with target "unix_oidc_audit", event_type "KEY_GENERATED", key_type "DPoP", and a non-empty key_id (thumbprint prefix, first 8 chars)
    - Test 2: `ProtectedSigningKey::from_key()` emits event_type "KEY_LOADED" with key_type "DPoP" and key_id
    - Test 3: Dropping a `ProtectedSigningKey` emits event_type "KEY_DESTROYED" with key_type "DPoP" and key_id
    - Test 4 (negative): The audit events do NOT leak the full key material — only the thumbprint prefix appears in the event
    - Test 5: When the daemon loads a key from storage at startup, the KEY_LOADED event is emitted (test via grep on tracing output)
  </behavior>
  <action>
    In `unix-oidc-agent/src/crypto/protected_key.rs`:

    1. In `generate()`, after the key is created and mlock'd, emit:
       ```rust
       tracing::info!(
           target: "unix_oidc_audit",
           event_type = "KEY_GENERATED",
           key_type = "DPoP",
           key_id = %thumbprint_prefix,
           "DPoP signing key generated"
       );
       ```
       Where `thumbprint_prefix` is the first 8 chars of the SHA-256 thumbprint. Compute the thumbprint BEFORE returning so the event has the identifier.

    2. In `from_key()`, after the key is wrapped and mlock'd, emit:
       ```rust
       tracing::info!(
           target: "unix_oidc_audit",
           event_type = "KEY_LOADED",
           key_type = "DPoP",
           key_id = %thumbprint_prefix,
           "DPoP signing key loaded from storage"
       );
       ```

    3. In the `Drop` impl, before zeroing, emit:
       ```rust
       tracing::info!(
           target: "unix_oidc_audit",
           event_type = "KEY_DESTROYED",
           key_type = "DPoP",
           key_id = %thumbprint_prefix,
           "DPoP signing key destroyed"
       );
       ```
       Note: The Drop impl must not panic. If thumbprint computation fails during drop, use "unknown" as the key_id.

    4. Add a helper method `fn thumbprint_prefix(&self) -> String` that returns the first 8 hex chars of the SHA-256 thumbprint of the public key. This method is used by all three audit event emission points. If this method already exists as `thumbprint()`, reuse it and take the first 8 chars.

    **Security constraint:** Never log the private key bytes or the full thumbprint in audit events. The 8-char prefix provides enough correlation for lifecycle tracking without being a useful fingerprint for attackers.

    **Testing:** Use `tracing-test` crate or `tracing_subscriber::fmt::TestWriter` to capture audit events in tests. Verify the event fields match expectations. Ensure no key material appears in the captured output.
  </action>

  </verify>
