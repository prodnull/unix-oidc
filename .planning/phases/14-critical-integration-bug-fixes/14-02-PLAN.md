<objective>
Implement the SSH_ASKPASS nonce handler subcommand that bridges PAM's keyboard-interactive DPoP nonce challenge to the agent daemon's GetProof IPC. Without this, SSH login with dpop_required=Strict always fails because no code intercepts the DPOP_NONCE: prompt and calls GetProof with the extracted nonce.

Purpose: This is the client-side half of SEC-05 (server-side DPoP nonce issuance). The PAM module issues nonces correctly (Phase 7), but the SSH client has no handler to complete the round trip.
Output: `unix-oidc-agent ssh-askpass` subcommand that handles all three PAM prompt types.
</objective>

- SSH spawns SSH_ASKPASS binary per prompt with prompt as argv[1]

- Each invocation is a separate process (no shared in-process state)
- All invocations share the same parent PID (the ssh client process)

    **Create `unix-oidc-agent/src/askpass.rs`:**

    New module implementing the SSH_ASKPASS handler. The function signature:
    ```rust
    pub async fn run_ssh_askpass(prompt: String) -> anyhow::Result<()>
    ```

    Logic:
    1. Compute nonce tmpfile path: `std::env::temp_dir().join(format!(".unix-oidc-nonce-{}", std::os::unix::process::parent_id()))`.
    2. If `prompt.strip_prefix("DPOP_NONCE:")` succeeds:
       - Extract nonce value (trim whitespace).
       - Write nonce to tmpfile with `std::fs::write()`.
       - Set file permissions to 0600 via `std::os::unix::fs::PermissionsExt::from_mode(0o600)`.
       - Print empty line to stdout (`println!()`).
       - Return Ok.
    3. If `prompt.trim_start().starts_with("DPOP_PROOF")`:
       - Read nonce from tmpfile (best-effort; None if file missing).
       - Delete tmpfile (best-effort).
       - Resolve target hostname from `UNIX_OIDC_TARGET` env var. If not set, fall back to empty string (the PAM module's DPoP htu claim carries the server hostname; the agent uses it to construct the proof).
       - Call `AgentClient::default().get_proof(&target, "SSH", nonce.as_deref()).await`.
       - On `AgentResponse::Success(AgentResponseData::Proof { dpop_proof, token, .. })`:
         - Write token to a tmpfile at `std::env::temp_dir().join(format!(".unix-oidc-token-{}", std::os::unix::process::parent_id()))` with 0600 perms (for round 3 retrieval).
         - Print `dpop_proof` to stdout.
       - On error: print error to stderr, exit 1.
    4. If prompt contains "OIDC Token" or "Token" (case-insensitive match for the PAM `get_auth_token()` prompt):
       - Try reading cached token from the token tmpfile (`.unix-oidc-token-{PPID}`).
       - If cached token exists, print it and delete tmpfile.
       - If not, call `AgentClient::default().get_proof(&target, "SSH", None).await` and print the token field.
    5. For any unrecognized prompt: print empty line to stdout and exit 0 (safe default; SSH will receive empty response).

    **Security considerations (document in code comments):**
    - Tmpfiles use 0600 permissions (owner-only). The nonce is ephemeral (60s TTL on server) and the token tmpfile lives only milliseconds between rounds.
    - Nonce value is not sensitive (server-generated challenge, not a secret), but restrict permissions anyway as defense-in-depth.
    - Token tmpfile contains the access token briefly; 0600 perms + immediate deletion mitigate exposure.
    - PPID keying prevents cross-session collision. PPID is the ssh client process PID, stable across all SSH_ASKPASS invocations within one connection.

    **Wire into main.rs:**
    - Add `mod askpass;` to main.rs (or lib.rs if applicable).
    - Add `SshAskpass` variant to `Commands` enum:
      ```rust
      /// Handle SSH keyboard-interactive prompts (invoked as SSH_ASKPASS).
      ///
      /// SSH spawns SSH_ASKPASS with the prompt string as argv[1].
      /// Configure: SSH_ASKPASS=unix-oidc-agent SSH_ASKPASS_REQUIRE=force
      #[command(name = "ssh-askpass")]
      SshAskpass {
          /// The prompt string from SSH keyboard-interactive conversation
          prompt: String,
      },
      ```
    - Add match arm: `Commands::SshAskpass { prompt } => askpass::run_ssh_askpass(prompt).await,`

    **IMPORTANT:** The `prompt` argument in the `SshAskpass` variant is passed by SSH as a positional argument (argv[1]), NOT as a flag. Use clap's positional argument syntax (no `#[arg(long)]`).
