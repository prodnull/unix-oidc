  <action>
    **Logrotate config** at `deploy/logrotate.d/unix-oidc`:
    ```
    /var/log/unix-oidc-audit.log /var/log/unix-oidc/*.log {
        weekly
        rotate 52
        compress
        delaycompress
        missingok
        notifempty
        create 0640 root adm
        sharedscripts
        postrotate
            # Signal syslog daemon to reopen log files if needed
            /usr/lib/rsyslog/rsyslog-rotate 2>/dev/null || true
        endscript
    }
    ```

    Key decisions in the config:
    - `weekly` + `rotate 52` = 1 year retention (SOC2/PCI-DSS minimum)
    - `compress` + `delaycompress` = gzip after 1 rotation (allows active tools to read current-1)
    - `create 0640 root adm` = new log readable by adm group, not world-readable (security: audit logs may contain usernames and IPs)
    - `missingok` = don't error if the log file doesn't exist yet
    - `notifempty` = don't rotate empty files

  <action>
    Write `docs/gdpr-erasure-guide.md` covering:

    **Section 1: Data Inventory**
    What data unix-oidc stores, organized by storage location:

    | Data Type | Storage Location | Retention | Contains PII |
    |-----------|-----------------|-----------|--------------|
    | OIDC access tokens | Agent memory (SecretString) | Session lifetime | Yes (sub, email claims) |
    | OIDC refresh tokens | Keyring or file storage | Until revocation | Yes |
    | DPoP private keys | Agent memory (mlock'd) | Session lifetime | No (ephemeral key material) |
    | DPoP private keys (persisted) | Keyring or file storage | Until rotation | No |
    | Session records | `/run/unix-oidc/sessions/` | Until session close or sweep | Yes (username, client IP) |
    | Audit logs | Syslog + `/var/log/unix-oidc-audit.log` | Per retention policy | Yes (username, source IP, issuer) |
    | Issuer health state | `/run/unix-oidc/issuer-health/` | Transient | No |
    | Config files | `/etc/unix-oidc/` | Indefinite | No (unless usernames in break-glass config) |

    **Section 2: Erasure Procedures**
    For each data type, step-by-step erasure instructions:

    - **In-memory data**: Automatic via `ZeroizeOnDrop` (tokens) and `mlock` + `munlock` (keys). No manual action needed. Data is zeroed when the agent process exits or the key is rotated.
    - **Keyring entries**: `unix-oidc-agent storage clear` or manual keyring deletion via `secret-tool` (Linux) / `security delete-generic-password` (macOS).
    - **File storage**: `unix-oidc-agent storage clear` triggers three-pass DoD 5220.22-M-style overwrite (NIST SP 800-88 Rev 1 note: on CoW/SSD, full-disk encryption is the true defense).
    - **Session records**: Automatically removed on session close or by the sweep task. Manual: `rm /run/unix-oidc/sessions/*`.
    - **Audit logs**: `shred -u /var/log/unix-oidc-audit.log` for local files. Note: syslog entries shipped to external SIEM cannot be erased by unix-oidc — the data controller must contact the SIEM vendor.
    - **Config files**: Remove any PII from break-glass account names if applicable.

    **Section 3: What Cannot Be Erased**
    Be explicit about limitations:
    - Syslog entries forwarded to external SIEM systems (Splunk, ELK, etc.) are outside unix-oidc's control
    - Kernel audit logs (if auditd captures PAM events) require separate erasure
    - IdP-side session/token records are managed by the identity provider, not unix-oidc
    - CoW filesystem snapshots may retain overwritten data (recommend full-disk encryption)

    **Section 4: Implementation Guidance for Data Controllers**
    - How to respond to a GDPR Article 17 erasure request for a specific user
    - Step-by-step checklist: 1) revoke tokens at IdP, 2) clear agent storage, 3) delete session records, 4) purge audit logs (local + SIEM), 5) verify erasure
    - Recommended automation: script template for bulk erasure
    - Note on Article 17(3)(b) exemption: audit logs required for legal compliance (SOC2, PCI-DSS) may be retained under legitimate grounds

    Reference: GDPR Articles 5(1)(e), 17(1), 17(3)(b); NIST SP 800-88 Rev 1 for media sanitization.
