          ```rust
          struct OcsfFields {
              category_uid: u32,    // 3 = Identity & Access Management
              class_uid: u32,       // 3002 = Authentication
              activity_id: u32,     // 1=Logon, 2=Logoff, 3=Change, etc.
              severity_id: u32,     // 1=Info, 2=Low, 3=Medium, 4=High, 5=Critical
              type_uid: u32,        // class_uid * 100 + activity_id
              metadata: OcsfMetadata,
          }
          ```

       b. OCSF field mapping per event type:

          | Event | activity_id | severity_id | Notes |
          |-------|-------------|-------------|-------|
          | SSH_LOGIN_SUCCESS | 1 (Logon) | 1 (Info) | Successful auth |
          | SSH_LOGIN_FAILED | 1 (Logon) | 3 (Medium) | Failed auth |
          | TOKEN_VALIDATION_FAILED | 1 (Logon) | 3 (Medium) | Invalid token |
          | AUTH_NO_TOKEN | 1 (Logon) | 3 (Medium) | No token provided |
          | USER_NOT_FOUND | 1 (Logon) | 3 (Medium) | Unknown user |
          | STEP_UP_INITIATED | 3 (Auth Challenge) | 1 (Info) | Step-up started |
          | STEP_UP_SUCCESS | 3 (Auth Challenge) | 1 (Info) | Step-up completed |
          | STEP_UP_FAILED | 3 (Auth Challenge) | 3 (Medium) | Step-up denied |
          | BREAK_GLASS_AUTH (alert) | 1 (Logon) | 5 (Critical) | Emergency access |
          | BREAK_GLASS_AUTH (no alert) | 1 (Logon) | 1 (Info) | Routine BG |
          | SESSION_OPENED | 1 (Logon) | 1 (Info) | Session start |
          | SESSION_CLOSED | 2 (Logoff) | 1 (Info) | Session end |
          | SESSION_CLOSE_FAILED | 2 (Logoff) | 3 (Medium) | Cleanup failure |
          | TOKEN_REVOKED | 2 (Logoff) | 1 (Info) | Token revocation |
          | INTROSPECTION_FAILED | 99 (Other) | 3 (Medium) | Token check |
          | ISSUER_DEGRADED | 99 (Other) | 4 (High) | IdP unhealthy |
          | ISSUER_RECOVERED | 99 (Other) | 1 (Info) | IdP recovered |

       c. In `log()`, create a wrapper struct that combines the event JSON with OCSF fields:
          ```rust
          #[derive(Serialize)]
          struct EnrichedAuditEvent<'a> {
              #[serde(flatten)]
              event: &'a AuditEvent,
              category_uid: u32,
              class_uid: u32,
              activity_id: u32,
              severity_id: u32,
              type_uid: u32,
              metadata: OcsfMetadata,
          }
          ```
          This uses `#[serde(flatten)]` to include all existing event fields without modification, then adds OCSF fields alongside. Fully backward compatible — existing fields unchanged.

       d. Update `log()` to serialize `EnrichedAuditEvent` instead of bare `self`.

    **Backward compatibility guarantee:** The `#[serde(flatten)]` approach means ALL existing JSON fields (event, timestamp, user, host, etc.) appear exactly as before. OCSF fields are additive new top-level keys. No existing SIEM integration breaks.
