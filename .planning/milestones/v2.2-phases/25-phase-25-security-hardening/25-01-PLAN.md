  <name>Task 1: Algorithm enum match (TryFrom) and configurable per-issuer allowlist</name>
  <files>pam-unix-oidc/src/oidc/validation.rs, pam-unix-oidc/src/policy/config.rs</files>
  <behavior>
    - Test: TryFrom converts KeyAlgorithm::RS256 to Algorithm::RS256 successfully
    - Test: TryFrom converts KeyAlgorithm::ES256 to Algorithm::ES256 successfully
    - Test: TryFrom rejects KeyAlgorithm variants not in the JOSE signing algorithm set (e.g. encryption algorithms) with Err
    - Test: When JWKS key has alg=RS256 and token header has alg=ES256, validation returns UnsupportedAlgorithm error (positive mismatch detection via enum, not serde)
    - Test: When JWKS key omits alg and token header claims HS256, validation rejects with UnsupportedAlgorithm (default allowlist blocks symmetric algorithms)
    - Test: When JWKS key omits alg and token header claims RS256, validation proceeds (RS256 is in default allowlist)
    - Test: When per-issuer allowed_algorithms is set to [ES256] only, a token with RS256 in header against a JWKS key without alg is rejected
    - Test: When per-issuer allowed_algorithms is set to [ES256], a token with ES256 proceeds
    - Test: IssuerConfig with allowed_algorithms field deserializes from YAML correctly
    - Test: IssuerConfig without allowed_algorithms field uses default allowlist (backward compat)
  </behavior>
  <action>
1. In `validation.rs`, create `impl TryFrom<jsonwebtoken::jwk::KeyAlgorithm> for jsonwebtoken::Algorithm` (or a local wrapper type if orphan rule prevents it) with exhaustive match arms for: RS256, RS384, RS512, ES256, ES384, PS256, PS384, PS512, EdDSA. All other KeyAlgorithm variants (encryption algs, HS*) return Err. This replaces the serde serialization comparison at lines 306-329.

2. Define `const DEFAULT_ALLOWED_ALGORITHMS: &[Algorithm]` containing RS256, RS384, RS512, ES256, ES384, PS256, PS384, PS512, EdDSA.

3. In `config.rs`, add `pub allowed_algorithms: Option<Vec<String>>` to `IssuerConfig` with `#[serde(default)]`. When None, use DEFAULT_ALLOWED_ALGORITHMS. Document the field with security rationale.

4. Replace the serde-based comparison block (lines 306-329) with: convert `jwk_alg` via TryFrom, then compare directly with `==` against the token header algorithm. The TryFrom conversion failure itself rejects unknown algorithms.

5. Replace the BLOCKED_ALGS blocklist (lines 331-353) with an allowlist check: when JWKS key omits `alg`, check `algorithm` against the configured (or default) allowed_algorithms list. Reject anything not in the list.

6. Thread the `allowed_algorithms` from `IssuerConfig` into `ValidationConfig` so the validator has access to the per-issuer list.

7. Ensure all tests use `cargo test --features test-mode -p pam-unix-oidc` for tests requiring insecure validator.
