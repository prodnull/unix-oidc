**Part A: Create test/docker/Dockerfile.test-host-systemd**

Create a Dockerfile based on `jrei/systemd-ubuntu:22.04` (a systemd-enabled Ubuntu image designed for Docker testing). This image supports `systemctl` inside a container.

```dockerfile
FROM jrei/systemd-ubuntu:22.04

# Install PAM development dependencies and SSH server
RUN apt-get update && apt-get install -y \
    libpam0g \
    libssl3 \
    openssh-server \
    curl \
    jq \
    && rm -rf /var/lib/apt/lists/*

# Copy unix-oidc-agent binary (provided at build time via ARG)
ARG AGENT_BINARY=target/release/unix-oidc-agent
COPY ${AGENT_BINARY} /usr/local/bin/unix-oidc-agent
RUN chmod +x /usr/local/bin/unix-oidc-agent

# Copy systemd unit files
COPY deploy/systemd/unix-oidc-agent.service /etc/systemd/user/unix-oidc-agent.service
COPY deploy/systemd/unix-oidc-agent.socket /etc/systemd/user/unix-oidc-agent.socket

# Enable lingering for testuser (allows user systemd units to run)
RUN useradd -m testuser && loginctl enable-linger testuser

# systemd entrypoint (provided by jrei/systemd-ubuntu)
CMD ["/lib/systemd/systemd"]
```

**Part B: Create test/tests/test_systemd_launchd_e2e.sh**

Create an executable bash script covering systemd E2E (CI) and launchd documentation (local macOS).

Variables: `COMPOSE_FILE="${COMPOSE_FILE:-docker-compose.systemd-e2e.yaml}"`, `SYSTEMD_SERVICE="test-host-systemd"`.

Script structure:

1. **Detect OS and CI context**: if `SKIP_SYSTEMD_TEST=1` is set, print SKIP and exit 0.

2. **Test 1 — systemd socket activation:**
   - Inside the systemd container: `docker compose exec "$SYSTEMD_SERVICE" sudo -u testuser systemctl --user daemon-reload`
   - `docker compose exec "$SYSTEMD_SERVICE" sudo -u testuser systemctl --user enable --now unix-oidc-agent.socket`
   - Wait 2 seconds.
   - `docker compose exec "$SYSTEMD_SERVICE" sudo -u testuser systemctl --user is-active unix-oidc-agent.socket`
   - Assert: exit 0 and output contains "active".
   - Send a Status IPC command to the socket: `docker compose exec "$SYSTEMD_SERVICE" sudo -u testuser unix-oidc-agent status 2>&1 | grep -q "Agent running\|status"`.
   - pass() on socket active + status response.

3. **Test 2 — JSON log format under journald:**
   - Start daemon: `docker compose exec "$SYSTEMD_SERVICE" sudo -u testuser systemctl --user start unix-oidc-agent`.
   - Wait 2 seconds.
   - Capture journald output: `docker compose exec "$SYSTEMD_SERVICE" sudo -u testuser journalctl --user -u unix-oidc-agent -n 20 -o json 2>/dev/null`.
   - Assert: output is valid JSON (pipe through `jq . > /dev/null`).
   - Assert: JSON contains `"MESSAGE"` field (journald JSON format).
   - pass() on assertion.

4. **Test 3 — Graceful shutdown:**
   - `START_TIME=$(date +%s)`
   - `docker compose exec "$SYSTEMD_SERVICE" sudo -u testuser systemctl --user stop unix-oidc-agent`
   - `END_TIME=$(date +%s)`
   - Assert: `ELAPSED=$(( END_TIME - START_TIME ))` and `[ "$ELAPSED" -lt 10 ]` (shutdown in <10s).
   - Assert: `docker compose exec "$SYSTEMD_SERVICE" sudo -u testuser systemctl --user is-active unix-oidc-agent` returns "inactive" or "failed".
   - pass() on timely shutdown.

5. **Test 4 — launchd install/uninstall (macOS local, skip in CI):**
   - If `uname` is not "Darwin", print: `echo "SKIP: launchd test — macOS only. Run locally: unix-oidc-agent launchd-install && launchctl list | grep unix-oidc-agent"`.
   - Otherwise: run `unix-oidc-agent launchd-install` and assert plist exists at `~/Library/LaunchAgents/com.unix-oidc.agent.plist`.
   - run `unix-oidc-agent launchd-uninstall` and assert plist removed.
   - pass() on both assertions.

6. Print summary.
