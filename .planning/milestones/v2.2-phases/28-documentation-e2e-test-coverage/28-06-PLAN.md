**Dockerfile.test-host changes:**

After the existing `RUN useradd -m -s /bin/bash testuser` line, add:

```dockerfile
# Create unix_oidc_users group for login_groups policy enforcement.
# testuser is a member (can authenticate via OIDC).
# testuser2 is a local system user but NOT a member (denied by login_groups policy in E2ET-02 Test 2).
RUN groupadd unix_oidc_users && \
    usermod -aG unix_oidc_users testuser && \
    useradd -m -s /bin/bash -G "" testuser2
# Note: testuser2 is created with no supplemental groups. It is explicitly
# NOT added to unix_oidc_users. This ensures NSS getgrouplist(testuser2)
# returns only the private group testuser2, never unix_oidc_users.
```

**policy-break-glass-e2e.yaml (new file):**

Create `test/fixtures/policy/policy-break-glass-e2e.yaml` with the following content:

```yaml
# test/fixtures/policy/policy-break-glass-e2e.yaml
# Policy fixture for E2ET-02: break-glass PAM bypass + NSS group policy denial.
#
# Key difference from policy-e2e.yaml:
#   ssh_login.login_groups restricts SSH to unix_oidc_users members only.
#   testuser is in unix_oidc_users (authenticated by Test 3).
#   testuser2 is NOT in unix_oidc_users (denied by Test 2).

host_classification: elevated

security_modes:
  jti_enforcement: warn
  dpop_required: warn
  amr_enforcement: warn
  acr_enforcement: warn
  groups_enforcement: warn

identity:
  username_claim: preferred_username

ssh_login:
  require_oidc: true
  minimum_acr: null
  max_auth_age: 3600
  login_groups:
    - unix_oidc_users

sudo:
  step_up_required: false

# Break-glass account bypasses OIDC and login_groups entirely.
# pam-unix-oidc returns PAM_IGNORE for break-glass users, falling through
# to pam_unix (local password auth). See CLAUDE.md §"Break-Glass Access".
break_glass:
  accounts:
    - breakglass
  require_otp: false
  alert_on_use: true

cache:
  jti_max_entries: 10000
  nonce_ttl_secs: 60

timeouts:
  clock_skew_staleness_secs: 60
  clock_skew_future_secs: 10
  jwks_cache_ttl_secs: 300
  http_timeout_secs: 10
```

**Why this approach:**
- `unix_oidc_users` is the NSS group name. `getgrouplist("testuser2", ...)` returns only the private group `testuser2` — it will never match the allowlist.
- `groupadd unix_oidc_users` creates the group in /etc/group inside the container. SSSD is not required for this check in test mode because `nix::unistd::getgrouplist()` reads from the NSS stack, which includes /etc/group.
- `alert_on_use: true` on break_glass is required for Test 1 to confirm the BREAK_GLASS_AUTH audit event is emitted at CRITICAL severity (per Phase 24-01 decision: `alert_on_use: true` triggers CRITICAL).
